# ADR 004: Timber Factory モジュール構成 (ロジックと UI の分離)

## 状況
スクリプトが肥大化し、特定の製品（障子）に依存した実装になっていた。将来的に他の木工製品（家具や建具）もサポートする場合、各製品ごとのロジックを管理しやすく、かつ共通の UI (ツールバーやエクスポート機能) を使い回す必要がある。

## 決定
プロジェクトの構成を「Timber Factory」という汎用的なフレームワークへとリファクタリングし、ロジックと UI を明確に分離する。

モジュール構造：
- **`timber_factory_main.py` (UI/Controller)**:
  FreeCAD のツールバー表示、パラメーターダイアログ、各種エクスポート処理を担当する。
- **`timber_logic_shoji.py` (Core Logic)**:
  障子の幾何学計算、あみだパターンの生成ロジックのみを担当する。
- **`exports/`**: 生成されたデータの出力先。
- **`docs/adr/`**: アーキテクチャ上の決定事項の記録。

## 影響
- 将来的な拡張性が大幅に向上し、新しい製品（例：`timber_logic_table.py`）を追加するだけで同様の UI とエクスポート機能を利用可能になった。
- 各モジュールのコードが整理され、メンテナンス性が向上した。
- ファイルの命名規則を構造化し、汎用的な名称を採用した。
- `src/` ディレクトリを導入し、ソースコードをプロジェクトルートから分離した。
