# ADR 002: 多層的エクスポート・フォールバック戦略 (GLTF/Mesh)

## 状況
FreeCAD 1.0 において、標準のエクスポートコマンド (`App.export`) が環境によってはエラーを返すケースがある。特に glTF エクスポートなどは、アドオンのインストール状況や環境変数に依存するため、すべての環境で動作する保証がない。

## 決定
複数のエクスポート方法を段階的に試行し、最も確実な方法でファイルを出力するフォールバック戦略を採用する。

エクスポートの優先順位：
1. **App.export (Native glTF)**:
   ネイティブのエクスポート機能を試行する。成功すれば、最も精度の高い出力が得られる。
2. **Mesh.export (Geometry Fallback)**:
   ネイティブ機能が失敗した場合、`Mesh` モジュールを使用して形状をテッセレーション (メッシュ化) し、それを GLTF 形式でエクスポートする。
3. **Mesh.export (OBJ Fallback)**:
   GLTF 形式自体の書き出しに失敗した場合は、最も汎用的な OBJ 形式としてメッシュ情報を出力する。

## 影響
- アドオンが欠落している環境でも、エラーで停止することなくエクスポートを完結させることができる。
- 3D Web プレビューなどの後続処理の成功率が大幅に向上した。
