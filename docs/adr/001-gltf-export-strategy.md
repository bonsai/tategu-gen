# ADR 001: FreeCAD 1.0 における GLTF エクスポート戦略

## ステータス
承認済み (Accepted)

## コンテキスト
FreeCAD 1.0 環境において、`App.export` (FreeCAD.export) が GLTF 形式を直接サポートしていない、またはアドオンが必要な場合にエラーが発生することが確認されました。
また、`Mesh.export` において拡張子 `.gltf` が認識されず、「Unknown extension」として失敗するケースがあります。

## 決定事項
GLTF エクスポート機能において、以下の多層的なフォールバック戦略を採用します。

1.  **App.export (標準)**: まず FreeCAD の標準エクスポートを試行します。アドオンが適切に設定されている場合はこれが最も正確です。
2.  **Mesh.export (フォールバック)**: `App.export` が失敗した場合、`Mesh` モジュールを使用してポリゴン化（テッセレーション）し、エクスポートを試みます。
3.  **拡張子エラーのハンドリング**: `.gltf` が認識されない場合、エラーメッセージを表示しつつ、フォールバックとして `.obj` (mesh_obj) 形式での出力を試行します。
4.  **ユーザーガイダンス**: 最終的に失敗した場合は、アドオンマネージャーから `glTF` アドオンをインストールするようユーザーに促します。

## 影響
- **利点**: ユーザーの環境設定（アドオンの有無）に関わらず、可能な限り 3D データを出力できます。
- **欠点**: `Mesh` モジュール経由のエクスポートでは、色やマテリアル情報が欠落する可能性があります。
